exactly — column names, types, constraints, defaults. Key constraints:

profiles: id uuid PK REFERENCES auth.users(id) ON DELETE CASCADE, role text CHECK
IN ('member','studio_owner','admin') DEFAULT 'member', credits integer NOT NULL
DEFAULT 0 CHECK (credits >= 0).

studios: slug text UNIQUE NOT NULL, owner_id uuid REFERENCES profiles(id) ON DELETE
SET NULL (nullable — allows studios without owner during seeding).

classes: studio_id uuid REFERENCES studios(id), credit_cost integer DEFAULT 1
CHECK (credit_cost > 0), difficulty_level CHECK IN
('beginner','intermediate','advanced').

bookings: status CHECK IN ('confirmed','cancelled','attended','no_show') DEFAULT
'confirmed', credits_charged integer NOT NULL.

credit_transactions: no DEFAULT for amount — it is required. type CHECK IN
('purchase','booking','refund','rollover','bonus').

subscriptions: status CHECK IN ('active','cancelled','past_due','trialing').

Do NOT use gyms, monthly_credits, gym_staff, or gym_id anywhere.

---

Create supabase/migrations/002_rls.sql. For each table: (1) ALTER TABLE ... ENABLE
ROW LEVEL SECURITY, (2) DROP POLICY IF EXISTS before each CREATE POLICY to be
idempotent. Use auth.uid() — NOT a custom auth.user_id() function.

Policies required:
- profiles: SELECT/INSERT/UPDATE own row (auth.uid() = id)
- studios: SELECT public read (is_active = true); INSERT/UPDATE only owner (auth.uid() = owner_id)
- instructors: SELECT public read (no filter); INSERT/UPDATE via studio owner
- classes: SELECT public read; INSERT/UPDATE via studio owner join
- bookings: SELECT own (auth.uid() = user_id); INSERT own; UPDATE own (status/cancelled_at only)
- credit_transactions: SELECT own; NO INSERT/UPDATE policy for authenticated role (service_role only writes)
- plans: SELECT public (is_active = true); no client INSERT/UPDATE
- subscriptions: SELECT own (auth.uid() = user_id); no client INSERT/UPDATE

CRITICAL: Do NOT add INSERT policy to credit_transactions for authenticated role.
The Update: Record&lt;string, never&gt; in types.ts enforces this immutability at the DB level too.
